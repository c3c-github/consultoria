Cronograma Detalhado do Projeto de BI

Plano de implementação da solução de Business Intelligence, do planejamento à manutenção.

1

Planejamento e Definição

Duração: ~1 semana

1. Reunião de Kickoff e Alinhamento de Requisitos

Reunião oficial com stakeholders para definir objetivos, KPIs e critérios de sucesso do MVP, garantindo alinhamento entre as áreas técnica e de negócio.

2. Mapeamento e Dicionário de Dados

Criar um documento que "traduz" cada evento técnico em termos de negócio para garantir consistência e evitar ambiguidades nas métricas.

3. Seleção dos Eventos e Categorias para o MVP

Com base nos KPIs e no dicionário de dados, selecionar formalmente o escopo de eventos que farão parte da primeira versão.

4. Criação de Protótipo do Dashboard Final

Desenhar um mockup de baixa fidelidade do dashboard para validação visual rápida com a área de negócio antes da construção técnica.

2

Implementação Técnica

Duração: ~1 semana

5. Setup de Infraestrutura e Acessos

Criar a conta de serviço, configurar permissões no Google Cloud (Firebase, BigQuery, Analytics) e ativar a exportação de dados.

6. Modelagem dos Dados no BigQuery (Views)

Criar `Views` (consultas salvas) no BigQuery para limpar, transformar e pré-calcular métricas, centralizando a lógica de negócio e otimizando a performance.

7. Validação de Custos e Performance

Executar as queries principais para validar o volume de dados processado e garantir performance aceitável.

8. Aprovação das Queries e Custos

Apresentar formalmente a projeção de custos e a lógica das queries para aprovação final da área de negócio.

3

Construção e Validação

Duração: ~1-2 semanas

9. Construção dos Dashboards

Conectar o Looker Studio às `Views` do BigQuery e construir os painéis conforme o protótipo aprovado.

10. Homologação do MVP

A área de negócio realiza testes nos dashboards para validar a correção dos dados, o funcionamento dos filtros e a aderência às expectativas.

4

Governança e Lançamento

Duração: ~1 semana

11. Configuração de Políticas de Retenção de Dados

Configurar a expiração automática de partições no BigQuery após 6 meses para controlar custos de armazenamento.

12. Configuração de Alertas de Custo e Monitoramento

Criar alertas de orçamento no Google Cloud Billing para notificar caso o uso se aproxime dos limites gratuitos.

13. Treinamento e Capacitação dos Usuários

Realizar uma sessão de treinamento com os usuários finais para ensinar a leitura dos dashboards, uso de filtros e extração de insights. Adoção é chave para o sucesso.

14. Lançamento Oficial (Go-Live)

Disponibilização dos links dos dashboards para todos os usuários autorizados.

5

Evolução e Manutenção

Contínuo

15. Suporte e Manutenção Corretiva

Plano de suporte para corrigir eventuais bugs ou inconsistências nos dados que possam surgir.

16. Backlog de Evolução

Criar e gerenciar um backlog com novas solicitações (novos eventos, gráficos, etc.) para planejar as próximas versões do produto.

17. Documentação Final do Projeto

Consolidar toda a documentação (dicionário de dados, lógica das métricas, links) para garantir a transferência de conhecimento e a longevidade do projeto.
