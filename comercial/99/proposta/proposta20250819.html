Proposta de Projeto: Acionamento e Conversão de Motoristas

Digitalização e Automação no Salesforce

Objetivo do Projeto

Digitalizar e automatizar o processo de acionamento e conversão de motoristas offline no Salesforce, substituindo o fluxo manual baseado em planilhas. O objetivo é criar um processo mais seguro, escalável e rastreável, fornecendo indicadores em tempo real para a gestão.

Escopo Funcional: O Que Vamos Construir

Hierarquia de Campanhas

Criação de campanhas "mãe" para a estratégia mensal e campanhas "filhas" automáticas por analista.

Carga de Leads via API

Flexibilidade para importar a base de motoristas via arquivo ou automaticamente via API padrão.

Disparo de WhatsApp

Analistas iniciam o contato via Digital Engagement com templates padronizados.

Transbordo com Omni-Channel

Respostas dos motoristas são roteadas em tempo real para o analista responsável.

Atualização de Conversão

Recebimento dos dados de abastecimento via API padrão para atualizar os leads.

Dashboards e Bonificação

Visão de performance em tempo real para todos e um relatório específico para o cálculo de bônus.

Visão Geral da Solução

1. Preparação: Time 99 cria a Campanha Mãe e carrega os Leads (via CSV ou API).
↓
2. Distribuição: Salesforce cria as Campanhas Filhas e distribui os Leads para os analistas.
↓
3. Acionamento: Analista envia template de WhatsApp via Digital Engagement.
↓
4. Interação: Se o motorista responde, a conversa é roteada via Omni-Channel para o analista.
↓
5. Conversão: Script Python da 99 envia os dados de abastecimento via API Padrão.
