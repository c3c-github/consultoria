Data Migration Plan (DMP)
Projeto 99 Abastece

Plano de Migração de Dados — 99 Abastece

Sem Migração Base Zerada Go-Live 2025

1. Introdução e Objetivo

O presente documento tem como objetivo formalizar o plano de migração de dados do projeto Salesforce 99 Abastece. Após análise conjunta entre as equipes da C3C Software e da 99, concluiu-se que não haverá migração de dados históricos para o ambiente Salesforce. O novo sistema iniciará com uma base limpa, recebendo apenas os dados provenientes das novas campanhas criadas após o Go-Live.

2. Escopo de Dados Existentes

Durante a fase de inception, foi identificado que as informações operacionais atualmente são mantidas em planilhas Google Sheets, scripts Python locais e relatórios manuais. Esses dados não serão migrados para o Salesforce, visto que:

  • Não há dependência funcional de históricos anteriores para a execução do novo processo.
  • As campanhas futuras e suas cargas de motoristas serão tratadas como novos registros.
  • A consolidação histórica poderá ser mantida pela 99 em suas fontes originais.

3. Conclusão da Análise de Migração

Decisão: Não haverá migração de dados históricos para o Salesforce.

Motivos:

  • O novo processo será iniciado exclusivamente com campanhas criadas a partir do Go-Live.
  • Os dados legados não são necessários para as regras de negócio da operação.
  • O design técnico prioriza performance e consistência, partindo de uma base limpa.
  • Qualquer migração futura será tratada como projeto separado, mediante novo mapeamento e validação.

4. Premissas e Riscos Associados

  • O ambiente Salesforce iniciará sem registros históricos.
  • Relatórios e dashboards apresentarão dados apenas a partir do Go-Live.
  • A equipe da 99 manterá os históricos originais em suas ferramentas atuais.
  • Em caso de necessidade futura de carga retroativa, será necessária uma nova fase de análise e testes.

5. Governança e Responsabilidades

EntidadeResponsabilidade
C3C SoftwareGarantir que a estrutura de dados Salesforce esteja preparada para receber novas cargas a partir do Go-Live.
99 (Unidade 99 Abastece)Manter os dados históricos em fontes originais e decidir sobre possíveis cargas futuras.
