3.2. Detalhamento: Definição da Estratégia do Mês
  • Processo:
    • Realizado nos últimos dias do mês anterior.
    • Baseado principalmente na análise dos resultados do mês corrente/anterior (planilha Excel, aba "Tracking").
    • Decisão principal: Qual base usar (Churn vs. Never Used) - Roberta verifica se há volume suficiente da base escolhida (normalmente Churn).
    • Ajuste de parâmetros da meta (Excel, aba "Meta [Mês]"):
      • Taxa de Conversão (%): Ajustada com base no desempenho anterior (se bateram muito acima da meta, pode aumentar). Ex: Passou de 15% para 10% quando a regra dos 7 dias foi implementada.
      • Contatos Diários por Analista: Pode ser ajustado para "forçar" maior volume e dificultar manipulação. Ex: Aumentou de 120 para 135 recentemente.
      • Volume Total de Contatos: Calculado com base na capacidade diária, nº de analistas e dias úteis no mês (contados a partir da disponibilização da base pela Roberta, no início do dia). Feriados e férias de analistas reduzem os dias úteis/nº de analistas no cálculo.
      • Gordura (%): Adiciona um percentual (ex: 20%) ao volume calculado para gerar a quantidade final de contatos a serem disponibilizados na planilha (para cobrir orgânicos, sem WhatsApp, etc.).
      • Regras de Bonificação: Faixas de % de conversão e valores/percentuais associados (parametrizável).
      • Data de Corte: Definição da data limite para registrar a 1ª abordagem (para garantir que haja tempo para conversão dentro do mês).
    • Outros Critérios Estratégicos (Input da Victoria/Negócio):
      • Cidades Foco: Atualmente, todas as cidades com 99 Abastece.
      • Níveis Prioritários (Level Driver): Atualmente, 2, 3 e 4. Nível 2 restrito a SP/RJ (decisão estratégica, pode mudar). Nível 1 não é trabalhado hoje. Importante: Roberta não filtra por nível ao preparar a base para a Adecco, apenas por Churn 30-60 (prioridade inicial).
      • Tipo de Veículo: Atualmente, apenas 4 rodas (carros). Alerta: Há planos futuros para incluir 2 rodas (motos), o que exigiria filtros adicionais.
  • Ferramentas: Planilha Excel (abas Tracking, Meta, Base de Contatos).
  • Responsáveis: Roberta (análise de dados, execução) e Victoria (decisão estratégica final, exceções de bonificação).
3.3. Coleta de Feedbacks (Processo Atual)
  • Analistas (Adecco) recebem feedbacks dos motoristas durante as conversas (ex: problemas com postos, dúvidas sobre app, elogios).
  • Feedbacks pontuais/críticos são enviados via print de WhatsApp para a Coordenadora (Thaís), que repassa para Roberta/Victoria.
  • Não há um processo formalizado de coleta/relatório consolidado de feedbacks pela Adecco.

4. Decisões e Alinhamentos

  1. Parametrização é Essencial: A solução Salesforce deve permitir que a equipe 99 (Roberta/Victoria) edite facilmente os parâmetros da "campanha" mensal (Taxa de conversão, Contatos diários, Volume total ou % de gordura, Faixas de bonificação, Data de corte, Níveis elegíveis, Cidades elegíveis).
  2. Visibilidade da Meta para Analistas: A equipe terceirizada (analistas e coordenadora) precisa visualizar os parâmetros da meta do mês dentro do Salesforce.
  3. Dashboard de Resultados: Os gráficos de acompanhamento que Roberta usa para apresentar resultados devem ser replicados como dashboards no Salesforce, acessíveis para Roberta/Victoria (não para a equipe Adecco).
  4. Conexão com Data Lake: Confirmada a impossibilidade de conexão direta Salesforce -> Data Lake 99 devido a restrições de segurança (matriz chinesa). A alternativa via Python API é o caminho a ser seguido.
  5. Carga Inicial vs. Diária: A primeira carga da base no mês (potencialmente grande) será via upload manual de arquivo (Excel/CSV gerado pela Roberta). As atualizações diárias (abastecimentos) serão via API Python -> Salesforce.
