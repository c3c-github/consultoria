1. Objetivos da Sessão

  • Entender a estrutura de dados do IFS (ERP) para viabilizar a integração.
  • Mapear a hierarquia Cliente > Contrato > Projeto no IFS.
  • Definir estratégias para sincronização de custos e receitas.

2. Discussões Principais

2.1. Estrutura no IFS

  • Hierarquia: No IFS, a estrutura é: Cliente -> Contrato -> Projetos. [cite_start]Um contrato pode ter vários projetos vinculados[cite: 16728].
  • Código de Projeto: A criação do código de projeto é manual e não segue um padrão rígido de "De/Para" com as linhas de negócio do Salesforce. [cite_start]Isso varia de cliente para cliente[cite: 16742].
    • [cite_start]
  • Itens de Faturamento: Os itens dentro do contrato no IFS representam os faturamentos e reajustes ao longo do tempo, e não necessariamente os produtos vendidos (SKUs) originais[cite: 16808, 16825].

2.2. Integração de Custos e Receitas

  • Visão de Custo: O custo real (material, mão de obra, etc.) é alocado no "Projeto" dentro do IFS. [cite_start]A C3C propôs trazer essa informação de volta para o Salesforce para dar visibilidade de margem ao comercial[cite: 17117, 17136].
  • Desafio de Detalhamento: O detalhamento profundo dos custos no IFS é complexo. [cite_start]A decisão foi trazer para o Salesforce os valores macro (Custo Total e Receita Total) por projeto, permitindo uma análise de rentabilidade sem sobrecarregar a integração com micro dados[cite: 17158, 17326].
    • [cite_start]
  • Snapshot Semanal: Definido que a atualização desses dados (Custo/Receita) no Salesforce pode ser semanal (toda sexta-feira), o que é suficiente para a gestão do Márcio[cite: 17393, 17395].

2.3. Padronização de Dados

  • Necessidade de Padronização: Para a integração funcionar, será necessário um esforço de padronização dos cadastros (Clientes e Linhas de Negócio) entre Salesforce e IFS. [cite_start]O Salesforce deve atuar como a origem mestre para dados comerciais[cite: 17235, 17564].

3. Decisões e Encaminhamentos

  • Fluxo de Integração: O fluxo ideal será: Salesforce envia dados da Venda -> Faturamento cria Contrato/Projeto no IFS -> IFS devolve Código do Projeto -> Integração atualiza Custo/Receita semanalmente no Salesforce.
  • Próximos Passos: Validar o processo manual de criação de contratos com o time de Faturamento (Luciane) para entender os campos obrigatórios.
